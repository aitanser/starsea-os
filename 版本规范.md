# 星海量子通用操作系统 版本规范

**生效版本**：全部版本  
**最后更新**：2027-01-31

---

## 一、版本号格式

```
<主>.<次>.<修订>[-<阶段>[.<序号>]]
```

| 部分       | 说明                                 |
| ---------- | ------------------------------------ |
| 主版本号   | 不兼容 API 或重大架构变更            |
| 次版本号   | 新增功能（向下兼容）                 |
| 修订号     | 问题修复（向下兼容）                 |
| 阶段       | `alpha`、`beta`、`rc`（小写）        |
| 序号       | 预发布版本必须带序号，从 `.1` 开始   |

**示例**：  
`1.3.0-alpha.1`、`1.3.0-beta.1`、`1.3.0-rc.1`、`1.3.0`

---

## 二、版本递进规则

| 变更                 | 版本变化                               | 示例                         |
| -------------------- | -------------------------------------- | ---------------------------- |
| 修复 Bug             | 修订号 +1                              | `1.2.0` → `1.2.1`            |
| 新增功能             | 次版本号 +1，修订号归零                | `1.2.0` → `1.3.0`            |
| 不兼容变更           | 主版本号 +1，次/修归零                 | `1.2.0` → `2.0.0`            |
| `alpha` → `beta`     | 阶段变，序号重置为 1                   | `1.3.0-alpha.2` → `1.3.0-beta.1` |
| `beta` → `rc`        | 阶段变，序号重置为 1                   | `1.3.0-beta.2` → `1.3.0-rc.1`   |
| `rc` → 正式版        | 去掉阶段标识                           | `1.3.0-rc.1` → `1.3.0`        |
| 同阶段迭代           | 序号 +1                                | `beta.1` → `beta.2`          |

---

## 三、打包与发布规则

| 版本类型 | 是否打包 | 说明                                   |
| -------- | -------- | -------------------------------------- |
| `alpha`  | ❌       | 仅源码，内部验证                       |
| `beta`   | ✅       | 公开测试，必须打包                     |
| `rc`     | ❌       | 源码验证，通过后转为正式版             |
| 正式版   | ✅       | 稳定版本，必须打包                     |

---

## 四、产品密钥绑定

- 每个**打包发布**版本（`beta` 和正式版）生成唯一的产品密钥和 bcrypt 哈希。
- `alpha` 和 `rc` 沿用上一个 `beta` / 正式版的密钥。
- 存储位置：
  - `system/config.py`（硬编码）
  - `version_keys.json`（外部）
  - `密钥.txt`（明文）
